## الملخص
تم إعادة تصميم الهيكل العام للواجهة الإدارية (AdminShell) إلى 5 مناطق ثابتة، لتوفير تجربة موحدة لجميع صفحات الإدارة.

## هيكل التخطيط الجديد

```
┌──────────────────────────────────────────────┐
│  Admin Top Bar (56px) — ثابت                  │
│  الاسم + الإشعارات + اللغة + المستخدم + خروج │
├──────────────────────────────────────────────┤
│  Action Toolbar (48px) — ثابت                 │
│  رجوع • تحديث • إضافة • تعديل • حفظ • طباعة  │
├──────────┬───────────────────────────────────┤
│ Sidebar  │  Main Workspace (يتمركز فقط هنا)   │
│ (260px)  │  المحتوى القابل للتمرير            │
│ ثابت     │                                    │
├──────────┴───────────────────────────────────┤
│  Bottom Status Bar (34px) — ثابت              │
│  الصفحة • رقم العملية • الحالة • الساعة       │
└──────────────────────────────────────────────┘
```

## القيود المعروفة

1. أزرار action bar (إضافة، تعديل، حذف، حفظ، إلغاء، طباعة، تصدير، بحث) معطلة
   لأن معالجات الصفحة الفردية لم تُربط بعد. رجوع وتحديث يعملان.
2. شريط الحالة السفلي يعرض بيانات وصفية كمكان (—) لأن سياق الصفحة لم يُوفر بعد.
   الساعة تعمل.
3. أيقونة الإشعارات مرئية فقط (لا يوجد عدد غير مقروء غير مزيف).
4. الأزرار الموجودة في الصفحات الفردية (مثل Create/Edit/Delete داخل DataTable)
   لم تُمس — تعمل كما كانت.
5. شريط الحالة يحتاج لربط بالصفحات لتوفير البيانات الوصفية الحقيقية
   (رقم العملية، createdBy، updatedBy، إلخ).
6. لا يوجد ESLint (موجود مسبقاً).
7. صور الـ screenshot تظهر شاشة التحميل (spinner) بدلاً من المحتوى الكامل
   لأن الصفحات محمية بالمصادقة ولا يوجد token أثناء الالتقاط.
