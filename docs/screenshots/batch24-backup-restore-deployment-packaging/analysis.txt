Documented Limitations
----------------------
1. Web login page 500 in dev mode: The /login page returns 500 in Next.js dev
   mode due to a stale webpack chunk. This is a pre-existing dev-mode issue;
   production build (:4000) compiles and runs correctly.

2. Windows services not installed: Service installation scripts are provided
   but were not executed, as this requires admin privileges and a production
   deployment environment.

3. SQL Server Express Edition: Backup compression disabled by default because
   the local SQL Server is Express Edition, which does not support WITH
   COMPRESSION. Use -NoCompression flag. Standard/Enterprise editions can
   omit this flag.

4. No actual production deploy performed: Deployment and rollback were tested
   in dry-run mode only. Full execution requires C:\ATsoftERP\App target and
   Windows service registration.
