## Corrective Actions (Commit c98b37f)

### Problem
The new `AuditService.log()` in modules/audit/audit.service.ts expected a single LogEntryDto object,
but many existing callers (roles.service.ts, machine-parts.service.ts, machine-documents.service.ts,
machine-categories.service.ts, barcode-scans.service.ts) passed 4-5 positional arguments as the old
common/audit/audit.service.ts did.

### Fix
- Made AuditService.log() accept both sigil: `string | undefined | { ...params }` as first argument
  with optional trailing positional args for backward compatibility
- Updated users.service.ts to use new object-param style
- common/audit/ now re-exports from modules/audit/ instead of duplicating
