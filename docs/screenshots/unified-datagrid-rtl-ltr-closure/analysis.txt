Analysis of codebase table implementations before the unified DataGrid migration.

=== Problem ===
Admin pages used DataTable from ui.tsx (lines 126-195). It was a minimal generic table:
- No RTL/LTR support
- No column filters
- No sort indicators
- Plain gray header (bg-gray-50) instead of professional dark green
- No actions dropdown — each page implemented inline action buttons inconsistently
- No alternating row colors
- No sticky header
- No filter row
- No per-column alignment control

=== Solution ===
Create AdminDataGrid component with:
- Dark green header (#1a5632) with white text
- RTL/LTR support via dir prop
- Column filters (toggleable filter row)
- Sort indicators with asc/desc cycling
- Actions dropdown per row (three-dot menu)
- Alternating row backgrounds
- Sticky header
- Per-column alignment control
- Loading/empty/error states
- Global search bar built in
- Refresh button built in
- Filter toggle button with visual filter count indicator
