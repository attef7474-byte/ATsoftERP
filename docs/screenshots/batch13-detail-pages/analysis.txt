===============================================
Date: 2026-07-16

Summary
-------
Created 10 production-grade operational detail page routes under the admin section.
Each detail page uses real GET-by-id API endpoints, displays full entity details
(code, name, status badge), metadata (created/updated timestamps), related records
via tabs, and workflow actions (activate/deactivate via PATCH with status field).

List Pages Updated
------------------
All 10 list pages now include a "View Details" button in the actions column
that navigates to the corresponding /[id] detail page.

  i18n keys corrected.
- User list page: Same activate/deactivate fix; i18n keys corrected.
- Branch/Department detail pages: i18n 'navigation.back' -> 'common.back'.
- Department detail page: Import paths corrected.
- Products/Machines detail pages: BarcodeLabel type -> any.
- Machines detail page: Extended Machine type for related data fields.
- Adjustments detail page: errors.saveFailed -> errors.createFailed.
