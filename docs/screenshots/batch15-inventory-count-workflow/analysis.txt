## Barcode Integration

The execute page calls the real /barcodes/scan endpoint with
purpose=INVENTORY_COUNTING and auto-adds the scanned product as a
count line. This is a real API integration, not a placeholder.

## Workflow States Covered

1. DRAFT -> start -> IN_PROGRESS
2. IN_PROGRESS -> execute (scan/manual add/count/verify)
3. IN_PROGRESS -> review (filter differences)
4. IN_PROGRESS -> approve/complete -> COMPLETED
5. COMPLETED -> adjust (generate adjustment) -> DRAFT adjustment
6. DRAFT adjustment -> post -> POSTED adjustment
7. DRAFT/IN_PROGRESS -> cancel -> CANCELLED

## Duplicate Prevention

Adjust page checks for existing non-cancelled adjustments before
allowing generation, preventing duplicate adjustments from the same count.
