ROOT CAUSE OF ACTIONS MENU HIDDEN UNDER SIDEBAR
================================================

The actions dropdown in AdminDataGrid was rendered using CSS `position: absolute`
inside a `relative` container within the table cell. The dropdown was clipped
by two issues:

1. Stacking context conflict:
   - `.admin-main` container (position: fixed, z-index: 40) creates a local
     stacking context in the root layout.
   - All children of `.admin-main`, including the actions dropdown, are stacked
     within this local context (z-index 40 in root).
   - The sidebar (`.admin-sidebar`) sits at z-index: 50 in the root stacking
     context, so it always renders above the dropdown regardless of the
     dropdown's internal z-index.

2. Overflow clipping:
   - The grid wrapper had `overflow: hidden`, which clipped the absolutely
     positioned dropdown when it extended outside the container bounds.

FIX STRATEGY
============

- Removed `overflow: hidden` from the outer grid container.
- Rendered the actions menu via a React portal (`createPortal`) attached to
  `document.body`, so the menu lives at the root stacking context level.
- Used `position: fixed` with coordinates computed from
  `button.getBoundingClientRect()`.
- Set z-index to 100 (above sidebar's z-index: 50, action bar's 55,
  and topbar's 60).
- Reposition menu on scroll and resize.
- Close on Escape key, outside click, or action selection.
- Prevent menu from going outside viewport.
- Handle RTL alignment correctly.

PAGES MIGRATED TO UNIFIED DATAGRID
====================================

All 7 pages now use AdminDataGrid with the same green header, white text,
sticky header, column filters, sort indicators, portal-based actions menu,
RTL/LTR support, and i18n labels.

LIMITATIONS
===========

- The portal approach requires a `mounted` state check to ensure `document.body`
  exists (avoids SSR hydration errors).
- The actions menu does not auto-close when the data refreshes, but the
  openMenuRow state will reset to null when the component re-renders with new
  data (useEffect dependency).
